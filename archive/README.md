# archive/ — spent v2.0 planning documents

This folder holds the **completed** planning and phase-implementation documents
for the v2.0 development cycle, retained for historical reference:

- **`PLANNED-FEATURES-v2.md`** — the v2.0 feature specs, design rationale, and
  flagged assumptions, written *before* the features were built.
- **`IMPLEMENTATION-PHASES.md`** — the dependency-ordered build plan: the nine
  v2.0 phases (F1–F9), each with its ready-to-paste session prompt, testing
  requirements, and suggested commit message.

## These are NOT active specs

All nine v2.0 phases shipped (see `STATUS.md` and the GDD Version History), and
each feature's spec has moved into `asteroid-field-deluxe-GDD.md` Section 2.
These two documents are **fully superseded** by the shipped game and the current
GDD. Do **not** treat anything here as a live specification or an open to-do
list — they describe the plan as it was, not the game as it is.

## Where the authoritative docs live

- **Current design + what's actually shipped** → `asteroid-field-deluxe-GDD.md`
  (Section 2 always describes the current build).
- **Build reality + session decisions** → `STATUS.md`.

Kept only so the reasoning behind v2.0 stays recoverable. When a v3.0 cycle
begins, its own planning docs live at the repo root (per the "After Phase 9"
note that was in `IMPLEMENTATION-PHASES.md`), not here.
