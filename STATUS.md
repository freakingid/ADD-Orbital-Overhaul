Before that: implemented **F9 — Achievements System**, the final v2.0 phase. Added an `Achievements` module holding two pools — **15 weekly** (a calendar-deterministic 5-wide slice active each ISO week) and **12 lifetime** (cumulative across games) — that **observes** stats the rest of the game already produces via one-liner hooks at each event's existing site (kills in `destroyDebris`/`destroyHunter`, deliveries in the dock-offload block, hits in `damageShip`, deflects at the four shield-absorb sites, powerups in `applyPowerup`, expiries in `Garbage.update`, perfect waves in the wave-clear block, best-wave in `nextWave`) — **no new indirection layers, no restructured systems**. Weekly rotation is pure math — pool index `(isoYear*52 + isoWeek) % 15`, 5 consecutive entries wrapping the pool, from the real UTC date at launch — so **every player worldwide sees the same 5 in the same calendar week**; a new week resets weekly unlocks while lifetime is always retained. An unlock pops a non-blocking gold **toast** (`AudioSys.achievement()` fanfare, aged out in the main loop by `updateToasts` so it animates in every state) and is browsable in a read-only **viewer at Pause → Options → Achievements** (`drawAchievements`: this week's 5 + the 12 lifetime, each with unlock status + `cur/goal` progress). `Achievements.evaluate()` runs once per frame (already-unlocked entries short-circuit) and again at game over (so end-of-game checks like Waste Not can fire).

## Known issues

- **Real-`localStorage` persistence is unverified across the board (v1.9 + v2.0).** Both the settings key (`afd_settings_v1`) and the new achievements key (`afd_achievements_v1`) round-trip headlessly against a stub, but real cross-session persistence can only be confirmed in a browser running the file locally (the claude.ai artifact sandbox has no `localStorage`). All access is `typeof`-guarded + try/caught, so the sandbox worst case is "progress just doesn't persist," never a crash. **Top persistence playtest item.**
- **A few starter achievements are intentionally hard or degenerate (v2.0), by design of the starter pool — not bugs.** *Waste Not* ("zero canisters expired in a game") is near-impossible to earn legitimately (a lineage sheds 39–66 canisters and the chain caps at 12), yet is trivially satisfiable by dying before anything decays — it's implemented literally per F9's list. Curating/balancing the pools (retiring degenerate ones, tuning thresholds) is a **post-playtest pass**, deliberately not done this phase.
- **Weekly achievements reset each calendar week (v2.0), by design.** Unlocking a weekly one records it only for the current ISO week; when the week rolls over the 5 change and weekly-unlock progress starts fresh (lifetime never resets). This is the intended "weekly challenge" model — flagged so a mid-week reset (which won't happen — the week only changes on Monday UTC) isn't mistaken for data loss.
- **Achievements viewer is reachable only from Pause → Options (v2.0).** Since the menu opens only by pausing a live game, you must start a game to browse achievements. The title screen shows a hint pointing there; a first-class title-screen entry is a noted easy follow-up (same as the F8 "menu reachable only via pause" note).
- **Gamepad rebind drops that action's analog axis (v1.9), by design.** Capturing reads a button, not a stick direction; Return to Defaults restores the axis.
- **Audio unlock on a controller-only start is browser-dependent (v1.8).** Best-effort `AudioSys.init()`/`resume()` on Start/A, but the Web Audio autoplay policy generally wants a keyboard/pointer gesture — a controller-only player may hear nothing (incl. the new achievement fanfare + menu blips) until one keyboard press.
- **Soft weapon cap (v1.7), by design:** a Triple volley can transiently exceed the 12-bullet cap by up to 2.
- **Performance watch (v1.4/v1.6):** heavy `shadowBlur`; garbage-dense late waves are the stress case (a cleared Hunter lineage sheds 66 canisters). `GARBAGE_DECAY` is the fastest lever, then a global glow toggle. The v2.0 achievement layer adds negligible per-frame cost (≈17 simple `cur()` compares, already-unlocked short-circuit).

## Balance notes

- **Achievements (v2.0) — thresholds are first-pass, no balance knobs beyond the four `ACH_*` constants.** `ACH_TOAST_TIME` (4.5 s), `ACH_WEEKLY_SHOWN` (5), `ACH_LINEAGE_FULL` (13 = a full Hunter lineage), `ACH_SAVE_EVERY` (30 s periodic save). The 27 achievement goal numbers came straight from F9's starter pools — they want a playtest read on whether they're *earnable but meaningful* (see playtest asks). - **Carried forward, unchanged:** menu `VOL_STEP` (0.1) and `GP_DEADZONE` (0.25) (v1.9/v1.8); `POWERUP_DROP_CHANCE` (0.10), `POWERUP_HEALTH_GAP` ([18,26]), `POWERUP_DURATION` (15), weapon caps (Rapid 8 / Triple 12), `MAGNET_*`, `ENGINE_MASS_MULT` (0.5) (v1.7); the Hunter tables + `HUNTER_FLOOR_FRAC` (0.58) + shared `RAMP_WAVES` (8) (v1.6/v1.5); `GARBAGE_DECAY` 12 s / `GARBAGE_SEVER_DECAY` 10 s, wave spawn `min(3+wave, 9)` (v1.4).

**Achievements (v2.0/F9):**
1. **Do settings AND achievements actually persist across a real reload?** This is the top ask. Earn an achievement (e.g. deliver 20 canisters → Scrap Runner, or destroy 15 debris → Satellite Buster), change some volumes/bindings, close the tab, reopen the file. Are the unlock, the lifetime counters, and the settings all still there? Two separate `localStorage` keys are in play (`afd_settings_v1`, `afd_achievements_v1`) — confirm both survive. *(Only confirmable outside the artifact sandbox — headless tests exercise only the serialize/deserialize logic.)*
2. **Does the unlock toast read clearly and feel good?** Non-blocking gold banner top-centre, ~4.5 s, fade in/out, stacks if several land at once, plus a rising-triad chime. Is it legible over a busy field? Too long/short? Does it distract during intense play (it's drawn on top of everything, including the pause menu)?
3. **Is the Achievements viewer legible?** Pause → Options → Achievements: this week's 5 (left) + the 12 lifetime (right), each with a ✓/`cur/goal`, earned rows in gold. 17 rows in one panel — does it feel cramped, and do the progress numbers read at a glance? Would you want a first-class title-screen entry (currently only the hint is there)?
4. **Do the achievement thresholds feel earnable-but-meaningful?** The 27 goals are first-pass from the starter pools. Any that feel unreachable, trivial, or degenerate (Waste Not especially — flagged as near-impossible-yet-cheesable)? This decides which to retune/retire in a curation pass.
5. **Sanity-check the "same 5 every week" premise informally** — the active weekly set is fixed by the ISO calendar week; it should be stable all week and change on Monday (UTC). No action needed unless it visibly misbehaves.

**Menu / options / rebinding (v1.9/F8):**

7. Does a **rebound control take effect immediately** in live play? Are the two quirks acceptable — a gamepad rebind **dropping that action's stick axis**, and **no conflict detection** when rebinding to an already-used key?

**Controller (v1.8/F7):** deadzone comfort at `GP_DEADZONE` 0.25; shield on RT/RB (a gap-fill — right button?); D-Pad-vs-stick feel; whether a controller-only start produces sound or needs one keyboard press to unlock audio (known Web Audio limitation).
